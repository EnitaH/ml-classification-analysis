Dataset Description¶

For this project, I decided to use the Breast Cancer Wisconsin (Original) dataset from the UCI Machine Learning Repository. This dataset was created by Dr. William H. Wolberg at the University of Wisconsin Hospital and is particularly designed as a benchmark for binary classification in medical diagnosis. I downloaded the dataset from the UCI repository on 6th November 2025. After researching several options, I found that this dataset was the most suitable, and there were several reasons that motivated me to work with it.

Firstly, it focuses on a medical classification problem that aims to distinguish between benign and malignant breast tumours using cytological measurements. This makes the task clinically meaningful and aligned with real-world healthcare challenges. Secondly, the dataset is relatively small, with 699 instances and nine predictive attributes (in addition to ID and class label). This size is appropriate for a machine-learning coursework project and allows me to explore multiple models and experiments efficiently. Finally, the dataset has been widely used in research, providing useful baselines for comparison and supporting the academic relevance of my work.

The features include clump thickness, uniformity of cell size, uniformity of cell shape, marginal adhesion, single epithelial cell size, bare nuclei, bland chromatin, normal nucleoli, and mitoses. The main target variable is a binary class label indicating whether the tumour is benign or malignant.

The purpose of my project is to use this dataset to build and evaluate machine-learning classifiers that predict tumour type based on these cytological features. I will explore the data, apply appropriate preprocessing such as handling missing values and scaling, and compare multiple traditional classification models. The final aim is to identify the best-performing model in terms of predictive accuracy and confusion-matrix performance, and to compare my results with findings reported in the research literature.

Literature Review¶

The paper by Asri et al. (2016) evaluates the performance of four traditional machine-learning classifiers—Support Vector Machine (SVM), k-Nearest Neighbours (k-NN), Decision Tree (C4.5), and Naïve Bayes—on the Wisconsin Breast Cancer dataset. Their objective is to identify which algorithm provides the most accurate diagnosis of benign versus malignant tumours. This aligns directly with the aim of my project, which is to compare multiple classical classifiers on the same dataset. The authors follow a clear methodology involving train-test splitting and evaluation using accuracy, precision, recall, and F1-score, mirroring the expected workflow for this coursework. Their findings show SVM achieving the highest accuracy (97%), followed by Decision Trees and Naïve Bayes. These results helped establish realistic expectations for my own models, motivating careful preprocessing and hyperparameter tuning to achieve performance within the 90–97% range. The paper is methodologically transparent and uses the identical dataset, making it a strong benchmark reference. As it is published in Procedia Computer Science, a peer-reviewed and widely cited venue, it represents a credible and relevant source for traditional ML approaches.

The second source, Wolberg, Street, and Mangasarian (1995), is foundational, written by the creators of the Wisconsin dataset itself. They propose computer-assisted diagnostic models based on fine-needle aspirate (FNA) cytology measurements, aiming to distinguish benign from malignant tumours with accuracy comparable to medical experts. Their methods use linear programming techniques such as the multisurface method–tree (MSM-T) and polynomial discriminant functions, differing from the traditional ML models used in my project but analysing the same feature set (e.g., cell size uniformity, bare nuclei, chromatin texture). Their reported accuracy (92–97%) further validates the performance range achieved in my experiments. As a highly credible, medically focused, peer-reviewed publication in Breast Cancer Research and Treatment, this paper provides authoritative background on the diagnostic value of the dataset’s features and supports the relevance of my modelling approach.

Best Model Discussion¶

Across all four models tested, the Support Vector Machine with an RBF kernel achieved the strongest overall performance, offering the highest test accuracy and the most balanced classification metrics. This aligns with findings from the literature, where SVMs frequently perform well on medium-sized biomedical datasets due to their ability to model non-linear relationships. The EDA showed correlated cytological features and non-linear separation tendencies, which SVM handles effectively. Its relatively low training error combined with strong generalisation suggests minimal overfitting.

Random Forest also performed competitively, particularly in recall for the malignant class. However, it demonstrated slightly higher variance between training and testing accuracy, indicating mild overfitting. Logistic Regression provided a strong baseline but underperformed compared to non-linear models, likely due to its linear decision boundary, which is insufficient for the complex structure of this dataset. k-NN performed reasonably but was sensitive to feature scaling and class distribution.

Overall, the SVM model best met the objectives of this mini-project by achieving high accuracy, balanced recall, and strong robustness across both classes. Its performance exceeded expectations based on prior research, and no other model demonstrated a superior balance of interpretability, stability, and predictive quality for this classification task.

Limitations¶

A key limitation of this project is the relatively small dataset size, which can restrict model generalisation and make performance sensitive to minor variations in train–test splits. Although stratification was used, the dataset remains mildly imbalanced, meaning recall for the malignant class must be interpreted cautiously. Another limitation is that only tabular cytology features were used; more complex clinical information or image-based features could lead to better predictive performance. Finally, hyperparameter tuning was limited to small, focused grids, so it is possible that more extensive searches could yield marginally higher accuracy.
